# Привет! Я Елена (elenadewind) 
## Мои проекты
Я разрабатываю инструменты для безопасности кода, локального ИИ и автоматизации юридической обработки документов. Сейчас активно веду три репозитория:

### 1. security-audit-checklist
Практический чек‑лист для аудита безопасности кода. Помогает находить уязвимости до релиза.

**Что умеет**:
- ищет опасные системные вызовы (`os.system()`, `eval`);
- проверяет права доступа (Linux/macOS/Windows);
- интегрируется со статическими анализаторами (Bandit, ShellCheck).

**Для кого**: разработчики, DevOps, аудиторы безопасности.
→ [GitHub](https://github.com/elenadewind/security-audit-checklist)

### 2. local-llm-runner
Инструмент для запуска LLM (Llama, Mistral и др.) локально — без облака и передачи данных наружу.

**Ключевые фишки**:
- работает на потребительских GPU;
- имеет API для интеграции в приложения;
- кэширует ответы для ускорения.

**Для кого**: разработчики, исследователи NLP, компании с требованиями к приватности.
→ [GitHub](https://github.com/elenadewind/local-llm-runner)

### 3. LegalDataProcessor‑Pro
Профессиональное решение для автоматизированного анализа юридических документов с поддержкой ИИ и API‑интеграций.

**Основные возможности**:
- анализ PDF, DOCX, TXT (включая OCR для сканов);
- проверка на соответствие нотариальным требованиям;
- интеграция с ERP‑системами (SAP, Oracle, 1C);
- веб‑интерфейс для нетехнических пользователей.

**Для кого**:
- юридические фирмы;
- корпоративные департаменты;
- нотариальные палаты.
→ [GitHub](https://github.com/elenadewind/LegalDataProcessor-Pro)


**Технологии**: Python, Flask, React, OCR (Tesseract), spaCy.  
**Лицензия**: проприетарная (см. LICENSE.md).

- #### 4. AI-Powered-Behavioral-Anomaly-Detector-v.1.0
- A malware detection system that uses AI to mask code. It analyzes the behaviour of processes rather than static code.
- Система обнаружения вредоносного ПО, использующего ИИ для маскировки кода. Анализирует поведение процессов, а не статический код.
- Key capabilities: Interception of calls to LLM APIs (Gemini, GPT, etc.).
- Analysis of prompt context for signs of malicious intent.
- Detection of anomalous behaviour (frequent code generations, injections).
- Local «ethical AI» for analysis without data leakage.
- Integration with SIEM/EDR systems.
- Ключевые возможности Перехват вызовов к LLM‑API (Gemini, GPT и др.).
- Анализ контекста промтов на признаки вредоносных намерений.
- Обнаружение аномального поведения (частые генерации кода, инъекции).
- Локальный «этичный ИИ» для анализа без утечки данных.
- Интеграция с SIEM/EDR‑системами.
